# 哈灵麻将 / 上海敲麻规则差距分析

更新时间：2026-07-01

对标来源：

- 哈灵麻将官网：https://haling.com/games/hlmj.html
- 上海麻将公开规则：https://zh.wikipedia.org/wiki/上海麻將

说明：哈灵官网是产品介绍页，不是完整规则页。它能确认哈灵麻将的产品定位、玩法集合和体验方向；具体胡牌、花数、辣子、包牌等细则仍需要 App 内规则页、对局录像或测试账号截图进一步校准。

## 已恢复的交互规则

- 手牌出牌规则已恢复为原来的习惯：第一次点牌选中，第二次点同一张牌直接出牌。
- 仍保留手牌区横向滚动、牌距增大、选中态提示等舒适性改动。
- 叫听时机已调整为：玩家打出一张牌后，如果剩余 13 张进入听牌，服务端立即向本人弹出叫听确认；不再等下一次轮到玩家摸牌/出牌时才提示。
- 叫听确认改为服务端重新校验当前 13 张手牌，避免客户端误报听牌。

## 当前已覆盖

- 144 张牌：万、条、筒、东南西北中发白、春夏秋冬梅兰竹菊。
- 中发白按花牌处理，摸到自动亮出并补牌。
- 不能吃牌。
- 支持碰、明杠、暗杠、加杠、胡、自摸。
- 听牌后只能打刚摸的牌。
- 碰杠胡响应优先级为：胡 > 杠 > 碰。
- 上局赢家当庄，首局随机庄。

## 和上海敲麻 / 哈灵体验的主要差距

### P0：玩法集合不完整

哈灵官网明确把产品定位为上海本地特色玩法集合，并点名包含“敲麻、清混碰、百搭”等多种玩法。当前项目只有一个固定的“上海敲麻 / 哈灵规则”模式，没有房间玩法选择，也没有“清混碰”“百搭”等独立规则配置。

建议：

- 大厅创建房间时增加玩法选择。
- 先实现 `敲麻` 和 `清混碰` 两个模式。
- `百搭` 作为后续模式，不要混进当前敲麻规则里。

### P0：胡牌口径偏宽

当前 `server.js` 的 `canHu()` 允许普通 4 面子 + 1 对的基础胡牌，还额外允许七对子、十三幺。

公开上海麻将资料通常强调可胡牌型更收窄，常见为混一色、碰碰胡、清一色、清碰、乱风向、字碰等。当前实现会让很多普通平胡也能胡，这和“上海敲麻”的防守节奏差距最大。

哈灵官网点名“清混碰”，说明至少需要把清一色、混一色、碰碰胡、清碰作为核心模式/牌型对齐。

### P0：缺少乱风向 / 字碰

当前没有独立识别：

- 乱风向：全风向牌成牌。
- 字碰 / 风碰：风牌刻子结构。

这会影响上海麻将标志性大牌型。

### P0：计分模型不是上海麻将“底 + 花 + 辣子”

当前计分为：

- `花数 * 2^番数`
- 封顶固定 `50`
- 点炮由放炮者付 `finalScore * 3`

公开上海麻将资料通常是底、花、辣子封顶、清一色/清碰/全风向等按辣子或倍数处理。当前番数模型更像通用麻将游戏化计分，不像哈灵/上海敲麻。

### P1：花数细则不完整

当前花数：

- 底花 1
- 每个补出的花牌 +1
- 明杠 +1，暗杠 +2

公开上海麻将资料里，东南西北、中发白的碰/杠花数通常有更细分的权重。当前因为中发白被直接补花，无法形成三元牌碰杠；东南西北碰出或暗刻、明杠、暗杠的花数也未完整体现。

### P1：缺少包牌 / 三口承包

公开上海麻将资料提到“三口包牌 / 承包”是特色规则。当前没有记录某玩家从另一玩家处吃碰杠次数，也没有包牌后的自摸、点炮结算变化。

### P1：缺少抢杠

当前加杠会直接转换为杠并进入杠后摸牌，没有给其他玩家检查抢杠胡的窗口。

### P1：八花报道 / 七捉一口径待校准

当前八花报道只统计春夏秋冬梅兰竹菊 8 张，不把中发白计入八花。公开资料里八花、七捉一常按约定变化。需要对齐哈灵 App 实际口径。

### P2：门清、敲、听牌产品表现还不完整

当前已修复“出牌后立即叫听”的核心时机问题，也保留了听牌后只能打刚摸牌的限制。但“敲”的表现仍更像听牌确认弹窗，不完全等同于哈灵麻将里的敲牌提示、状态展示和后续限制。

### P2：文档与实际实现不一致

`DEPLOY.md` 写“万、条、筒各36张，共108张”，但实际服务端已经是 144 张含字牌和花牌。需要同步文档，避免测试和运营认知冲突。

### P2：移动端产品体验距离哈灵还有差距

哈灵官网强调快节奏、流畅体验、智能选牌、一指出牌、微信快捷登录/邀请。当前项目已有房间号、二维码、浏览器分享、自动摸牌、出牌倒计时，但还缺：

- 智能选牌 / 推荐出牌。
- 微信快捷登录。
- 自动分桌 / 快速开局。
- 更接近移动端原生棋牌 App 的桌面密度、按钮层级和局内反馈。
- 玩法选择和规则说明入口。

## 建议修复顺序

1. 增加创建房间玩法选择：先支持 `敲麻` / `清混碰`，并把规则配置传到服务端。
2. 确定哈灵 App 的目标房间规则：是否允许平胡、七对子、十三幺，辣子封顶是几花，八花是否含中发白。
3. 修改 `canHu()` 和听牌检测，让服务端和前端都使用同一套“上海敲麻合法胡牌型”。
4. 重做计分：底、花、辣子、门清、清一色、清碰、全风向、字碰。
5. 补抢杠和包牌。
6. 做智能选牌 / 推荐出牌，贴近哈灵官网强调的一指出牌体验。
7. 更新游戏内规则页、`DEPLOY.md` 和测试用例。
