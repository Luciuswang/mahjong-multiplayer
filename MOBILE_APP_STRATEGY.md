# iOS / Android App Strategy

更新时间：2026-07-01

## 对标结论

对标对象：

- 哈灵麻将官网：https://haling.com/games/hlmj.html
- App Store：https://apps.apple.com/cn/app/id1426313974
- 应用宝：https://sj.qq.com/appdetail/wxa29b61784d4bfc3b

哈灵麻将不是单一麻将局，而是“本地休闲棋牌大厅”：

- App Store 标题为“哈灵麻将-本地休闲棋牌”，定位是“好朋友一起玩”。
- iOS 端是独立 iPhone App，免费、含内购、18+、App 体量约 566.5 MB。
- 玩法包含上海敲麻、红中血流、血战到底、杭州麻将、广东麻将、宁波麻将等地方麻将，以及斗地主、掼蛋、跑得快等扑克玩法。
- 官网明确宣传“敲麻、清混碰、百搭”等上海本地玩法集合。
- 应用宝链接偏微信小游戏入口，强调扫码马上玩、手机和电脑跨端畅玩。
- 近期版本重点是极速出牌、单击出牌、男女声语音切换、托管、免费玩法、好友系统、快速开始、反馈系统、场景/UI升级。

## 我们当前基础

已有：

- Web 单页游戏大厅。
- Socket.io 多人房间。
- 房间号、二维码、链接邀请。
- 聊天、快捷语、表情。
- 离线重连、AI接管、托管式超时出牌。
- 音频播报、震动反馈。
- 移动端 viewport、安全区适配。

缺口：

- 没有 iOS / Android 原生工程。
- 没有 App 图标、启动图、商店截图、隐私政策、用户协议、年龄分级材料。
- 没有微信/Apple/手机号账号体系。
- 没有玩法大厅和快速开始。
- 规则仍是“上海敲麻简化版”，不够像哈灵的多玩法产品。
- iOS 上如果只是 WebView 包网页，存在审核风险。

## 推荐技术路线

### Phase 1：内部测试 App

目标：最快拿到可安装的 iOS / Android 包，给团队和小范围用户试玩。

方案：

- 使用 Capacitor 包现有 Web 游戏。
- iOS 用 TestFlight，Android 先出 APK/AAB 内测。
- WebSocket 后端部署到 HTTPS/WSS 域名。
- App 内加载本地 Web 资源或固定远端 HTTPS 入口。
- 接入基础原生能力：启动图、横竖屏限制、分享、震动、音频权限、深链加入房间。

注意：

- 这一阶段适合内测，不建议直接冲 App Store 公测。
- 如果远端 WebView 痕迹太重，iOS 审核风险较高。

### Phase 2：商店可审版本

目标：让 App 看起来和用起来都是完整移动棋牌 App。

必须补：

- 原生启动页、登录页、设置页、隐私/用户协议页。
- App 内规则页和玩法选择页。
- 快速开始、好友房、最近一起玩。
- 智能选牌 / 推荐出牌。
- 单击出牌 / 二次点击出牌开关。
- 推送通知：好友邀请、房间召回、活动公告。
- 崩溃和性能监控。
- 明确的账号删除入口和隐私授权管理。

### Phase 3：哈灵式棋牌大厅

目标：不只是一个上海敲麻 App，而是可扩展的本地棋牌平台。

建议玩法顺序：

1. 上海敲麻。
2. 清混碰。
3. 百搭。
4. 斗地主 / 掼蛋等扑克玩法。

系统顺序：

1. 玩法大厅。
2. 快速开始。
3. 好友系统升级。
4. 战绩统计。
5. 活动和赛场。
6. 会员/装扮/音效等商业化模块。

## Capacitor 落地任务

当前项目满足 Capacitor 的基本条件：有 `package.json`，也有 `index.html`。但 Capacitor 更适合有独立构建产物目录，因此建议新增 `www` 或 `dist` 输出目录，而不是直接把整个仓库作为 `webDir`。

建议任务：

1. 新增前端构建脚本，把 `index.html`、`img`、`audio`、必要静态资源复制到 `www`。
2. 安装 Capacitor 依赖：`@capacitor/core`、`@capacitor/cli`、`@capacitor/ios`、`@capacitor/android`。
3. 新增 `capacitor.config.json`，建议包名先用 `com.codex.mahjong` 或公司正式域名反写。
4. 新增 iOS / Android 原生工程。
5. 配置 App 深链：`mahjong://room/{roomCode}` 或 HTTPS Universal Link / App Link。
6. 配置生产服务地址：`https://your-domain.example` + `wss://your-domain.example`。
7. 替换 CDN 依赖，减少 App 首屏受外网影响。

## 商店与合规清单

iOS：

- Apple Developer 账号。
- Bundle ID。
- App Store Connect 元数据。
- 隐私政策 URL。
- 账号删除入口。
- 年龄分级建议 18+。
- 如果接微信登录，需同时考虑 Apple 对第三方登录的要求。
- 避免“只是网页套壳”，需要 App 化功能和完整内容。

Android：

- Android 包名和签名证书。
- APK/AAB 构建。
- 国内分发渠道材料。
- 应用权限说明。
- 隐私政策和用户协议。

国内游戏合规：

- 需要确认是否涉及版号、ISBN、ICP备案、文网文/软著等要求。
- 哈灵官网展示了 ICP、ISBN、审批文号、文网游备案等材料。我们正式上架前也需要法务确认。

## 第一版 App MVP 范围

MVP 不追求一次做成哈灵体量，先做“能安装、能邀请、能稳定打一局”的 App：

- App 启动页。
- 昵称/头像登录。
- 创建房间 / 加入房间。
- 分享链接 / 扫码加入 / 深链回到房间。
- 上海敲麻一套规则。
- 语音播报、震动、托管、重连。
- 规则页。
- 反馈入口。

不放进 MVP：

- 真实货币相关。
- 复杂商城和内购。
- 多地区玩法全集。
- 大型活动和赛场。

## 下一步建议

1. 先做 Capacitor 内测包，把当前 Web 游戏打进 iOS / Android。
2. 同时把服务端部署到稳定 HTTPS/WSS 域名。
3. 补规则大厅和玩法配置，为“敲麻 / 清混碰 / 百搭”做结构准备。
4. 再按商店审核要求补隐私、账号、反馈、深链、推送、图标、截图。
